# Связь фронтенда и API

## Простая схема

```
┌─────────────────┐         HTTP запросы          ┌──────────────────┐
│   Фронтенд      │ ────────────────────────────> │   API (FastAPI)  │
│  (Браузер)      │                                │   (Сервер)       │
│                 │ <──────────────────────────── │                  │
│  HTML/CSS/JS    │      JSON ответы              │  Python модули   │
└─────────────────┘                                └──────────────────┘
                                                           │
                                                           ▼
                                                   ┌──────────────┐
                                                   │  ClickHouse  │
                                                   │   (БД)       │
                                                   └──────────────┘
```

## Зачем нужен API?

### 1. **Логика генерации в Python модулях**

Вся логика генерации написана на Python:
- `generators.py` - генераторы данных
- `profile.py` - работа с профилями
- `client.py` - подключение к ClickHouse

**Без API**: Пришлось бы переписать всю логику на JavaScript (дублирование кода)

**С API**: Используем существующие Python модули, ничего не дублируем

### 2. **Безопасность**

- Пароли и подключения к БД не попадают в браузер
- Вся работа с БД происходит на сервере
- Клиент не имеет прямого доступа к ClickHouse

### 3. **Переиспользование**

API можно использовать:
- Из веб-интерфейса (фронтенд)
- Из других приложений
- Из командной строки (curl, Postman)
- Из других языков программирования

## Как это работает?

### Пример 1: Предпросмотр данных

**1. Пользователь нажимает "Сгенерировать 10 значений"**

**2. Фронтенд (app.js) отправляет запрос:**
```javascript
const response = await fetch('/api/generate', {
    method: 'POST',
    body: JSON.stringify({
        generator_kind: 'random_int',
        generator_params: { min: 0, max: 100 },
        rows: 10,
        preview_only: true  // ← только предпросмотр
    })
});
```

**3. API (server.py) обрабатывает:**
```python
# Создает генератор из Python модуля
generator = build_generator('random_int', {'min': 0, 'max': 100})

# Генерирует 10 значений
for i in range(10):
    value = generator.next(i)
    results.append(str(value))

# Возвращает JSON
return {"success": True, "data": results}
```

**4. Фронтенд получает ответ и отображает:**
```javascript
result.data.forEach((value, index) => {
    // Показывает в таблице
    html += `<tr><td>${index + 1}</td><td>${value}</td></tr>`;
});
```

### Пример 2: Генерация в БД

**1. Пользователь нажимает "Применить к полю" и заполняет форму**

**2. Фронтенд отправляет запрос:**
```javascript
await fetch('/api/generate', {
    method: 'POST',
    body: JSON.stringify({
        generator_kind: 'random_int',
        generator_params: {...},
        connection: {
            host: 'localhost',
            port: 18123,
            username: 'default',
            password: 'ch_pass'
        },
        target_table: 'test',
        rows: 1000,
        preview_only: false  // ← генерация в БД
    })
});
```

**3. API обрабатывает:**
```python
# Создает профиль
profile = Profile.load(...)

# Подключается к ClickHouse
service = ClickHouseService(profile)

# Генерирует и вставляет данные
for row_index in range(1000):
    row = [generator.next(row_index)]
    service.insert_rows([row], column_names)
```

**4. Фронтенд получает результат:**
```javascript
// Показывает сообщение об успехе
showStatus('success', `Успешно сгенерировано ${result.rows_inserted} строк`);
```

## Что происходит без API?

Если бы не было API, пришлось бы:

1. **Переписать всю логику на JavaScript**
   - Дублирование кода из `generators.py`
   - Разные результаты (Python vs JavaScript)
   - Сложнее поддерживать

2. **Прямое подключение к БД из браузера**
   - Пароли в коде (небезопасно!)
   - CORS проблемы
   - Нужен публичный доступ к БД

3. **Нет переиспользования**
   - Только веб-интерфейс
   - Нельзя использовать из других приложений

## Преимущества текущей архитектуры

✅ **Единая логика** - один код для предпросмотра и БД  
✅ **Безопасность** - пароли на сервере, не в браузере  
✅ **Переиспользование** - API можно использовать откуда угодно  
✅ **Простота поддержки** - изменения в одном месте (Python модули)  
✅ **Консистентность** - предпросмотр показывает те же данные, что будут в БД  

## Файлы и их роли

- **`frontend/index.html`** - интерфейс (формы, кнопки)
- **`frontend/app.js`** - логика фронтенда (отправка запросов к API)
- **`backend/server.py`** - API сервер (обработка запросов)
- **`ch_synth/generators.py`** - логика генерации (используется API)
- **`ch_synth/client.py`** - работа с ClickHouse (используется API)

## Итого

**API - это мост между фронтендом и Python модулями:**

- Фронтенд = интерфейс пользователя (что видит пользователь)
- API = обработка запросов (что происходит на сервере)
- Python модули = бизнес-логика (как генерируются данные)

Без API пришлось бы дублировать всю логику на JavaScript, что неэффективно и небезопасно.
