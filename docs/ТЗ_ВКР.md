# Техническое задание на разработку приложения Field Generator

**Выпускная квалификационная работа**

---

## 1. Описание проекта

### 1.1. Цели проекта

**Цель:** Разработка программного средства для генерации синтетических тестовых данных и загрузки их в колоночные и реляционные СУБД с целью автоматизации подготовки тестовых наборов данных при разработке и тестировании информационных систем.

**Задачи:**
- Обеспечить генерацию данных различных типов (числа, строки, даты, UUID, regex-паттерны) с настраиваемыми параметрами
- Реализовать поддержку ClickHouse и PostgreSQL
- Предоставить веб-интерфейс и CLI для гибкой настройки схемы и объёма генерации
- Обеспечить возможность нагрузочного тестирования СУБД (бенчмарк IOPS)

### 1.2. Назначение и область применения

**Назначение:** Field Generator предназначен для автоматической генерации синтетических данных при:
- разработке и отладке приложений, работающих с базами данных;
- нагрузочном и интеграционном тестировании;
- наполнении тестовых и демонстрационных окружений;
- прототипировании отчётов и дашбордов.

**Область применения:**
- Разработка информационных систем (backend, аналитика, BI)
- Тестирование производительности СУБД (ClickHouse, PostgreSQL)
- Обучение и демонстрация работы с колоночными БД
- Наполнение data lake и хранилищ данных тестовыми наборами

**Целевая аудитория:** разработчики, тестировщики, аналитики данных, DevOps-инженеры.

## 1.3. Концепция проекта

### 1.3.1. Общие положения

Field Generator — веб-приложение с REST API и командным интерфейсом, позволяющее:
1. Описать схему таблицы (поля, типы, генераторы)
2. Настроить параметры генерации (количество строк, размер батча, режим по длительности)
3. Загрузить сгенерированные данные в ClickHouse или PostgreSQL
4. Выполнить предпросмотр без записи в БД

**Ключевые принципы:**
- Расширяемость: добавление новых типов генераторов без изменения ядра
- Многополевой режим: таблица с произвольным числом колонок
- Поддержка нескольких СУБД через единый интерфейс

### 1.3.3. Описание доменной модели приложения

**Основные сущности:**

| Сущность | Описание |
|----------|----------|
| **Profile** | Конфигурация генерации: подключение к БД, целевая таблица, список полей |
| **Field** | Поле таблицы: имя, тип (UUID, Int32, String, DateTime и др.), генератор |
| **Generator** | Алгоритм генерации значения: kind (random_int, uuid4, regex и др.), params |
| **Connection** | Параметры подключения к БД: host, port, username, password, database |

**Типы генераторов:**
- `random_int` — случайные целые/вещественные в диапазоне
- `sequence_int` — последовательность с опциональной вероятностью
- `timestamp_asc` / `timestamp_desc` — монотонные временные метки
- `uuid4` — UUID v4
- `random_digits` — строка цифр заданной длины
- `regex` — строка по регулярному выражению (пресеты: паспорт РФ, телефон, MAC)
- `enum_choice` — выбор из списка (случайно или по очереди)
- `url_template` — URL с подстановкой {row}, {uuid}

### 1.3.4. Структура базы данных

Приложение не хранит собственных данных; целевые таблицы создаются в СУБД пользователя.

**Пример схемы таблицы в ClickHouse (MergeTree):**
```sql
CREATE TABLE default.example_table (
    `id` UUID,
    `ts` DateTime,
    `value` Int32,
    `passport` String,
    `phone` String
) ENGINE = MergeTree
ORDER BY (id)
```

**Конфигурационные данные приложения:**
- `dictionaries/index.json` — именованные словари для enum_choice
- `profiles/*.json` — примеры профилей для CLI

### 1.3.5. Описание процессов работы системы (TO BE)

**Процесс 1: Генерация через веб-интерфейс**
1. Пользователь добавляет поля, выбирает типы и генераторы
2. Настраивает подключение к БД и имя таблицы
3. Указывает объём (количество строк или длительность × скорость)
4. Нажимает «Сгенерировать в БД»
5. Система создаёт таблицу (если не существует), генерирует данные батчами и вставляет в СУБД
6. Отображается предпросмотр вставленных данных

**Процесс 2: Генерация через CLI**
1. Пользователь подготавливает JSON-профиль (connection, target, fields)
2. Запускает `python -m ch_synth.cli --profile profile.json --rows N --batch-size M --create-table`
3. Система создаёт таблицу и вставляет N строк батчами по M строк

**Процесс 3: Бенчмарк IOPS**
1. Запускается ClickHouse (в т.ч. с лимитом памяти для стресс-теста)
2. Запускается `python scripts/benchmark_iops.py` с параметрами
3. Система измеряет throughput (строк/сек) при различном числе процессов

### 1.3.6. Пример интерфейса

**Веб-интерфейс:**
- Левая панель: список полей таблицы (имя, тип, генератор)
- Правая панель: параметры выбранного поля (min/max, regex, словарь и т.д.)
- Нижняя панель: подключение к БД, имя таблицы, объём генерации, кнопка «Сгенерировать»
- Блок предпросмотра: таблица с результатами
