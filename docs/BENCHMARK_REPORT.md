# Отчёт бенчмарка IOPS

**Дата:** 2025-02-13  
**Среда:** Windows, Docker ClickHouse (localhost:18123), multiprocessing

---

## Термины

| Термин | Пояснение |
|--------|-----------|
| **IOPS** | Строк вставлено в ClickHouse за секунду (inserts/sec) |
| **Batch** | Сколько строк накапливается перед одной отправкой в ClickHouse |
| **Light-профиль** | 2 поля: id (UUID) + value (Int32), ~20 байт/строка |
| **Полный профиль** | 16 полей: UUID, DateTime, Int64, Float64, regex (паспорт, телефон, MAC), enum, url и др., ~220 байт/строка |

---

## 1. Multiprocessing vs Threading

Переход с `threading` на `multiprocessing` (обход GIL):

| Режим | 4 процесса/потока | Прирост |
|-------|-------------------|---------|
| Threading | ~3 480 IOPS | — |
| Multiprocessing | **~8 140 IOPS** | **2.3×** |

---

## 2. Стресс-тест: Light-профиль (1 GB RAM)

Лимит ClickHouse: **1 GB**. Параметры: 100k строк/процесс, batch 50k.

| Процессов | IOPS | Строк | Статус |
|-----------|------|-------|--------|
| 1 | 89 096 | 100 000 | OK |
| 2 | 192 336 | 200 000 | OK |
| 4 | 356 521 | 400 000 | OK |
| 8 | 565 001 | 800 000 | OK |
| 16 | 787 844 | 1 600 000 | OK |
| 32 | **867 215** | 3 200 000 | OK |
| 64 | — | — | **OOM** |

**Максимум до падения:** 32 процесса, **867 215 IOPS**.

## 3. Стресс-тест: Полный профиль (1 GB RAM)

| Процессов | IOPS | Строк | Статус |
|-----------|------|-------|--------|
| 1 | 3 544 | 100 000 | OK |
| 2 | 6 688 | 200 000 | OK |

| 16 | — | — | **OOM** |

**Максимум до падения:** 8 процессов, **22 737 IOPS**.

## 4. Прямой запуск: 8 процессов (полный профиль)

```bash
python scripts/benchmark_iops.py --mode multi --rows 100000 --batch 50000 --threads 8
```

| Метрика | Значение |
|---------|----------|
| Строк | 800 000 |
| Время | 36.66 с |
| IOPS | **21 820** |
| На процесс | ~2 800 IOPS |

---

## 5. 32 процесса с отловом ошибок (--catch-errors)

При 32 процессах полный профиль даёт OOM. С `--catch-errors` считаем частичный результат:

```bash
python scripts/benchmark_iops.py --mode multi --rows 100000 --batch 50000 --threads 32 --catch-errors
```

| Метрика | Значение |
|---------|----------|
| Строк (частично) | 2 300 000 |
| Время | 83.78 с |
| IOPS | **27 452** |
| Процессов с OOM | 10 из 32 |

Часть процессов успела вставить данные до падения.

---

## 6. Сравнение Light vs Полный (1 GB)

| Профиль | Макс IOPS до падения | Процессов | Разница |
|---------|----------------------|-----------|---------|
| Light (id + value) | **867 215** | 32 | — |
| Полный (16 полей, regex) | 22 737 | 8 | **38×** |

Light даёт ~38× больше IOPS: меньше данных, проще генерация (без exrex).

---

## 7. Лимиты памяти и падение

| Лимит | Результат |
|-------|-----------|
| 128 MB | ClickHouse не стартует / OOM при старте |
| 256 MB | OOM при 1 процессе (полный профиль) |
| 1 GB | См. таблицы выше |

---

## Команды для запуска

```bash
# Обычный тест (4 процесса)
python scripts/benchmark_iops.py --rows 10000 --batch 1000 --threads 4

# Light-профиль (высокий IOPS)
python scripts/benchmark_iops.py --light --rows 50000 --batch 10000 --threads 8

# Стресс до падения
docker compose -f docker-compose.yml -f docker-compose.stress.yml up -d
python scripts/benchmark_iops.py --kill

# 32 процесса с отловом OOM
python scripts/benchmark_iops.py --mode multi --rows 100000 --batch 50000 --threads 32 --catch-errors

# Очистка таблиц bench_*
python scripts/benchmark_iops.py --cleanup-only
```

---

## Выводы

1. **Multiprocessing** даёт ~2.3× прирост по сравнению с threading.
2. **Light-профиль** — до 867k IOPS при 1 GB; полный — до ~23k IOPS.
3. **Узкое место** — память ClickHouse: при 1 GB полный профиль падает уже на 8–16 процессах.
4. **--catch-errors** позволяет получить частичный IOPS при OOM (27k при 32 процессах).
